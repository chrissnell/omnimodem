# The reference TUI client

`omnimodem-tui` ([`../../clients/omnimodem-tui/`](../../clients/omnimodem-tui/)) is a
Go terminal frontend and a complete worked example of driving the gRPC API. It is
not required by the daemon ‚Äî it is one client among any number a developer might
build. Paths below are relative to `clients/omnimodem-tui/`.

## What it is

A Bubbletea (Charmbracelet) terminal UI with a DOS-flavored theme. Entry point:
`cmd/omnimodem-tui/main.go`. It dials the daemon over gRPC ‚Äî a Unix-domain socket by
default (`$OMNIMODEM_RUNTIME_DIR/omnimodem.sock`, else
`<tempdir>/omnimodem/omnimodem.sock`), or an explicit `--addr` (a UDS path or a
`host:port` for the routable TCP endpoint). The Go gRPC bindings are committed
(`internal/pb/`) and regenerated by `gen.sh` (`make proto`) when the proto changes.
The operator's station identity (call/grid) and QSO log are local client state; all
modem configuration lives in the daemon.

## How it uses the gRPC API

| RPC | Where | Purpose |
|---|---|---|
| `GetState` | `msgs.go` (snapshot on connect / refresh) | Load channels into the UI. |
| `ListDevices` | `view_config.go` | Populate the device pickers. |
| `ConfigureChannel` | `view_config.go` | Persist name/mode/RSID edits. |
| `ConfigureAudio` | `view_config.go` | Bind RX/TX audio devices. |
| `ConfigurePtt` | `view_config.go` | Set PTT method + delay/tail. |
| `KeyPtt` | `view_operate.go` | Manual key (beacon/test). |
| `SetAudioGain` | `view_config.go` | RX/TX gain. |
| `ConfigureSpectrum` | `view_operate.go`, `waterfall.go` | Turn the waterfall on when entering Operate (audio passband) or the SDR view (`enableRFSpectrumCmd`, full RF span). |
| `SetSdrTune` | `sdr.go`, `view_sdr.go` | Absolute demod tune (step / direct entry); daemon splits into hardware center + NCO offset. |
| `SetSdrGain` | `sdr.go`, `view_sdr.go` | AGC toggle or manual tuner gain stepped through the caps table. |
| `ConfigureSdr` | `sdr.go`, `view_sdr.go` | Demod mode, squelch threshold, ppm correction. |
| `GetSdrCaps` | `sdr.go`, `view_sdr.go` | Tuner freq range + gain table, fetched on entering the SDR view. |
| `SuggestUdevRule` | `view_config.go` | PTT-permission troubleshooting helper. |
| `AcquireTxLease` / `ReleaseTxLease` | `tx.go` | Take/release exclusive TX around a transmission. |
| `Transmit` | `tx.go` | Queue a text/packet payload. |
| `TransmitImage` | `tx.go`, `picture_send.go` | Queue a raster (SSTV/WEFAX/Hell). |
| `SubscribeEvents` (stream) | `events.go` | The live feed: snapshot then `rx_frame`, `audio_level`, `ptt_state`, `clock_offset`, `channel_metrics`, `spectrum_frame`, `rsid_detected`, `channel_configured`, `sdr_state`. |

The channels view routes by bound device: a channel on an `rtltcp:<host>:<port>`
capture device (`isSDRDevice`) opens the **SDR tuning view**; every other channel
opens Operate. The SDR view is authoritative-daemon-driven ‚Äî tune/gain/demod/squelch
state is folded from the `sdr_state` event into `chanLive` (like `audio_level`), and
the view holds only editing state (step size, direct-entry buffer, gain-table
cursor, pending ppm). Keys: `‚Üê/‚Üí` tune ‚àì step, `s` cycle step (1k/5k/12.5k/25k), `f`
direct MHz entry, `g` AGC toggle, `[`/`]` step manual gain, `m` cycle demod, `,`/`.`
squelch, `\` squelch on/off, `-`/`+` ppm. The RF waterfall reuses the shared renderer
with a demod-channel **cursor** column (`waterfall.cursorColumn` maps Hz‚Üíbin via the
RF-referenced `freq_start_hz`/`freq_step_hz` that Plan 2 emits).

The Operate view adapts to the mode's *shape* (from `modes.go`): a **ragchew**
transcript + macros for keyboard modes, an FT8/FT4/JT65/JT9 **sequencer** ladder for
the slotted WSJT-X modes (with a UTC-anchored slot clock and QSO logging on RR73), a
**beacon** monitor for WSPR, and a scrolling **facsimile raster** for Hell/WEFAX ‚Äî
all sharing the waterfall renderer.

## UX terms

- **Waterfall** ‚Äî scrolling spectrum from `spectrum_frame` events; colour ramp per
  bin intensity, newest line at the bottom.
- **Macro** ‚Äî F1‚ÄìF5 templates with `{mycall}`/`{call}`/`{rst}` substitution, expanded
  and keyed at TX time.
- **Mode settings card** ‚Äî the Configure-view form whose fields are declared
  per-mode in `mode_settings.go` (center Hz, CW wpm/tone, RTTY baud/shift, FSQ
  directed flag, ‚Ä¶).
- **Sequencer / slot** ‚Äî auto-QSO ladder for time-windowed modes (FT8 15 s, FT4
  7.5 s, JT65/JT9 60 s, WSPR 120 s), aligned to UTC so stations stay in step.
- **Lease** ‚Äî the exclusive TX lock the client holds around a `Transmit` /
  `TransmitImage`, released on completion (see [`grpc-edge.md`](grpc-edge.md#tx-arbitration-interlock-vs-lease)).
