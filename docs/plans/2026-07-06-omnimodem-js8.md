# Phase W5 — JS8 (JS8Call) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Read the **Porting Doctrine** and the **T1–T9 per-mode task template** in `docs/plans/2026-07-02-omnimodem-full-mode-parity-implementation.md` before starting, plus the W5 line in `docs/plans/2026-07-02-omnimodem-fldigi-mode-parity.md`.

**Goal:** Port **JS8Call's JS8** into omnimodem, bit-exact-compatible on the air with the reference. JS8 is an 8-GFSK, 79-symbol windowed mode built on an *early FT8* core — **LDPC(174,87) / CRC-12** (not FT8's current (174,91)/CRC-14) — with a **75-bit message** (12 × 6-bit alphabet chars + a 3-bit frame type), three 7×7 Costas sync arrays, and the `S7 D29 S7 D29 S7` frame. The four submodes **Normal / Fast / Turbo / Slow** (and the disabled **Ultra**) are one parametric family differing only in samples-per-symbol / tone-spacing / T-R period / which Costas variant. On top of the modem sits JS8's application layer: the **JSC compressed-text codec** (a 262,144-entry dictionary — the bulk of the work) and the **directed protocol** (heartbeat/CQ, compound & directed frames, ACK/SNR, relay). The whole phase ships as one PR (T9).

**Architecture:** Two independent new building blocks, each KAT-gated in isolation before any mode uses it (Doctrine §6):

1. `crates/dsp/src/fec/ldpc_js8.rs` (+ `js8_tables.rs`) — the LDPC(174,87) code as an `Ldpc` instance built from generator + parity (`Nm`) tables transcribed **verbatim** from `js8call/lib/ft8/ldpc_174_87_params.f90`, plus **CRC-12** (poly `0xc06`, augmented). Reuses the existing BP decoder (`fec::ldpc::Ldpc`) and OSD (`fec::osd::osd_decode`) unchanged — only new tables. Gated against `ldpcsim174js8a` codeword output (Task 0).
2. `crates/dsp/src/framing/jsc.rs` (+ generated dictionary data) — the JSC compressed-text codec, ported verbatim from `js8call/{jsc.cpp,jsc_map.cpp,jsc_list.cpp}`. Pure integer/bit domain → **bit-exact** KAT vs the reference `JSC::compress`/`decompress`. Independent of the modem; gated on its own golden vectors (Task 1).

The message/framing layer `crates/dsp/src/framing/js8_message.rs` ports the 75-bit pack (`genjs8.f90` + `varicode.cpp`'s frame packers) and the directed-protocol frame types. The mode assembly `crates/dsp/src/modes/js8.rs` is one file carrying the shared 8-GFSK waveform, the `S7 D29 S7 D29 S7` sync/data frame, the two Costas variants, and a parametric `Js8Params` over the submode grid; it surfaces as `BlockDemodulator` + `Modulator` and wires into `mode::registry` like the Phase-W1 windowed modes, plus the TUI `modes.go` (T8, its own **directed** interaction shape).

**Tech Stack:** Rust (edition 2021, workspace). Reuses `omnimodem-dsp` blocks: `fec::ldpc::Ldpc` (BP min-sum), `fec::osd::osd_decode`, `fec::llr` (8-FSK soft demap), `frontend::modulate::Gfsk`, the FT8-style Goertzel spectrogram + Costas-array sync in `modes::ft8`/`sync::costas_array::CostasCorrelator`, and the conformance harness (`crates/dsp/tests/{kat.rs,ber.rs,loopback.rs}`, `testutil` AWGN/Watterson). Reference vectors come from the C++/Fortran already in-tree under `js8call/`.

## Reference facts (extracted — do not re-derive)

Pinned from `js8call/` at the workspace commit (record the exact hash — `a7ff1be` at time of writing — in the vector provenance at T1):

### Modem (`lib/js8/`, `commons.h`)

- **Frame** (`genjs8.f90:40-50`): `NN=79` symbols = `NS=21` sync (three 7×7 Costas) + `ND=58` data. Layout `S7 D29 S7 D29 S7`: `itone[0:7]=costasA`, `itone[7:36]=first 29 data`, `itone[36:43]=costasB`, `itone[43:72]=next 29 data`, `itone[72:79]=costasC` (0-based). 8-GFSK.
- **Data→tone map** (`genjs8.f90:46-49`): `indx = cw[3j]·4 + cw[3j+1]·2 + cw[3j+2]` — **plain binary MSB-first, NOT Gray** (differs from FT8's `genft8`). Port this mapping verbatim; the demod must invert the same plain-binary map (confirm against `js8dec.f90` at T5).
- **Costas variants** (`genjs8.f90:22-32`): `icos=1` (NCOSTAS=1, **Normal only**) uses the *original* array `A=B=C=(4,2,5,6,1,3,0)`. `icos=2` (NCOSTAS=2, **Fast/Turbo/Slow/Ultra**) uses three *symmetrical* arrays `A=(0,6,2,3,5,4,1)`, `B=(1,5,0,2,3,6,4)`, `C=(2,5,0,6,4,1,3)`. (Cross-check `lib/js8/costas/Costas_symmetrical_N=7.txt`.)
- **Waveform** (`genjs8refsig.f90`): continuous-phase FSK at `fs=12000`, per-symbol phase advance `dphi = 2π(f0·dt + itone/NSPS)`. TX uses GFSK pulse shaping (BT per FT8 family = 2.0; confirm the exact pulse at T4 against the reference `.wav`).
- **Submode grid** (`commons.h`, `lib/js8/js8{a,b,c,e,i}_params.f90`), `fs=12000`, all share `KK=87, ND=58, NS=21, NN=79`:

  | Submode | `SubmodeType` | NCOSTAS | NSPS (samples/sym) | Tx sec | Start delay | Tone spacing | Baud | RX SNR thr |
  |---|---|---|---|---|---|---|---|---|
  | Normal (A) | 0 | 1 (original) | 1920 | 15 | 500 ms | 6.25 Hz | 6.25 | −24 |
  | Fast (B) | 1 | 2 (sym) | 1200 | 10 | 200 ms | 10.0 Hz | 10.0 | −22 |
  | Turbo (C) | 2 | 2 (sym) | 600 | 6 | 100 ms | 20.0 Hz | 20.0 | −20 |
  | Slow (E) | 4 | 2 (sym) | 3840 | 30 | 500 ms | 3.125 Hz | 3.125 | −28 |
  | Ultra (I) | 8 | 2 (sym) | 384 | 4 | 100 ms | 31.25 Hz | 31.25 | −18 |

  Derived (from `JS8Submode.cpp`): `toneSpacing = 12000/NSPS`, `bandwidth = 8·12000/NSPS`, `framesForSymbols = 79·NSPS`, `framesPerCycle = 12000·txSec`. **Ultra is disabled in JS8Call's calling code** — port it as a defined-but-not-registered submode (mirror the reference), so the TUI grid exposes Normal/Fast/Turbo/Slow only unless the reviewer says otherwise.

### FEC (`lib/ft8/ldpc_174_87_params.f90`, `lib/ft8/encode174.f90`, `lib/crc12.cpp`)

- **LDPC(174,87)**: `N=174, K=87, M=87`. Generator `g(87)` — 87 rows of 22-hex-char strings (88 bits, top 87 used → 87 parity columns). Same table format as `ldpc_fst4.rs`'s generator; transcribe verbatim. Parity: `Mn/Nm/nrw` from the same params file (Tanner graph for BP). `colorder(174)` present for OSD.
- **CRC-12** (`lib/crc12.cpp`): boost `augmented_crc<12, 0xc06>` over an 11-byte buffer. In `genjs8.f90:35`: build the 75 message bits, zero the CRC field, compute `icrc12 = crc12(bytes,11)`, then **`icrc12 = xor(icrc12, 42)`** before appending. Port the augmented-CRC semantics + the `xor 42` exactly.
- **Message bits** (`genjs8.f90:33-42`): `cbits = 12×6-bit alphabet words (72) + i3bit (3) + crc12 (12) = 87`. Alphabet (68 chars): `0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz-+/?.`. Encoder maps each of 12 message chars to its 0-based index in that alphabet.

### Application layer (`jsc.cpp`, `jsc_map.cpp`, `jsc_list.cpp`, `varicode.cpp`, `varicode.h`)

- **JSC codec** (`jsc.cpp`): dictionary of `size = 262144` words.
  - `compress(text)`: split on spaces (KeepEmptyParts); for each word, greedily consume the **longest dictionary-prefix match** via `lookup`, emit `codeword(index, appendSpace, b=4, s=7, c=2^4−7=9)`. `codeword` emits a variable-length sequence of 4-bit groups (plus a 1-bit separator flag on the terminal group): `v=((index%s)<<1)+separate` as `b+1=5` bits, then higher base-`c` digits `+s` as `b=4` bits each, prepended. Returns `(bitvec, char_count)` pairs.
  - `decompress(bits)`: reads 4-bit groups; a group `<s` is terminal and reads 1 separator bit; reconstructs the index from base-`c` digits + `base[]` offsets; looks up `map[index].str`. Port `base[8]` and the reconstruction exactly.
  - `lookup(word)`: first-byte `prefix[103]` jump table → linear scan of that bucket in `list[]`, first `strncmp`-prefix match wins. **`list` order is load-bearing** (determines longest-match) → must be preserved from `jsc_list.cpp`.
  - `map[i]` = `{str, len, index=i}` (index-ordered dictionary; `decompress` + codeword index). `list[i]` = `{str, len, index→map}` (search order; `compress`). `prefix[i]` = `{first_byte, count, list_start}`; latin1 (not UTF-8) strings.
  - **Note the reference's last map entry** `{"ROSIDS", 1, 262143}` has an anomalous `len=1`; transcribe it as-is (it is a filler and its `len` mismatch is part of the reference behavior). Single-char and control/latin1 entries occupy the low indices (see `map[0..103]`).
- **Directed protocol** (`varicode.h:35-76`): 3-bit `i3bit` frame types — `FrameHeartbeat=0 [000]`, `FrameCompound=1 [001]`, `FrameCompoundDirected=2 [010]`, `FrameDirected=3 [011]`, `FrameData=4 [10X]`, `FrameDataCompressed=6 [11X]` (Data/DataCompressed encode only the top 2 bits, dropping the LSB, so they span two 3-bit codes each). `TransmissionType` first/last flags `[001]/[010]` mark message boundaries across multi-frame transmissions. The JSC codec feeds `FrameData`/`FrameDataCompressed`; the other frame types carry packed callsigns/grids/commands (`varicode.cpp` `pack*Message`/`unpack*Message`).
