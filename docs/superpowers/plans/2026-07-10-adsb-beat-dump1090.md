# ADS-B: Match-or-Beat dump1090/readsb — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise omnimodem's ADS-B decoder from a clean baseline (PR #111) to at least parity with — and ideally past — dump1090/readsb on real off-air 1090 MHz I/Q, proven by a reproducible frame-yield benchmark.

**Architecture:** Everything is gated by a **benchmark harness** built first: a fixed recorded I/Q file is replayed offline to both our decoder and the reference (dump1090/readsb), and we count valid-CRC frames + unique aircraft. Each decoder improvement is a hypothesis that must *move that number* on the recording or it is reverted. Improvements reuse existing `omnimodem-dsp` building blocks (`frontend::iq`, `frontend::resample`, `frontend::detector::EnvelopeDetector`, `frontend::agc`, `ensemble::ParallelDemodulator`) rather than reinventing them — the same "hydra" philosophy that made graywolf's AFSK beat Direwolf.

**Tech Stack:** Rust, `omnimodem-dsp` crate; `rtl-sdr` tools (`rtl_sdr`) for capture; dump1090-fa / readsb as the reference decoders.

---

## Success definition (read first)

We are "as good or better" when, on the **same** recorded I/Q file:

- **Primary metric:** our count of valid-CRC Mode S frames ≥ the reference's, and unique ICAO addresses ≥ the reference's.
- **Guardrail (no cheating):** false positives stay near zero. A frame we emit that the reference (and a union-of-all-decoders "ground truth") never sees, and whose ICAO never appears in a CRC-clean frame, is a false positive. Error-correction phases must not inflate the primary metric by manufacturing frames.
- **Reproducible:** a short committed clip runs the whole comparison in CI so we never regress.

Non-goal: live daemon RX (that is Phase 2 of `docs/design/2026-07-10-adsb-mode-design.md`). The benchmark is purely offline, so this plan does **not** block on the daemon wideband path.

## File structure

- Create `crates/dsp/src/modes/adsb/bench.rs` — pure decode-stats over a magnitude slice (`DecodeStats { frames, unique_icao, ... }`); reusable by tests and the bin.
- Create `crates/dsp/src/bin/adsb_bench.rs` — thin CLI: read a `uint8` I/Q file, convert, run, print stats. Ad-hoc use on big captures.
- Create `crates/dsp/src/modes/adsb/detect.rs` — noise-floor-relative preamble detector (R2).
- Modify `crates/dsp/src/modes/adsb/ppm.rs` — pluggable detector; phase-ensemble scan (R3); soft-decision slicing (R4).
- Modify `crates/dsp/src/modes/adsb/crc.rs` — syndrome error-correction table + 1-bit repair (R5).
- Create `crates/dsp/src/modes/adsb/roster.rs` — recently-seen ICAO set for repair/overlay corroboration (R5).
- Modify `crates/dsp/src/modes/adsb.rs` — wire detector/ensemble/roster into `AdsbDemod`.
- Create `crates/dsp/tests/data/adsb-clip.iq8` — small committed real-signal clip (a few hundred ms) for the CI gate.
- Modify `docs/design/2026-07-10-adsb-mode-design.md` — record the measured results per phase.

## Task 0: Ground-truth capture (blocked on hardware — run in parallel)

This produces the fixed input every later task measures against. **The operator runs the capture on the Pi;** we consume the file.

- [ ] **Step 1: Record raw I/Q on the Pi.** `rtl-sdr` package installed (`sudo apt install rtl-sdr`), dongle + tuned antenna connected. Record ~120 s of 1090 MHz at 2.4 Msps, high gain, as interleaved unsigned-8-bit I/Q (the `rtl_tcp`/dump1090 native format):

```sh
# 2.4 Msps x 120 s = 288e6 complex samples = 576 MB of uint8 I/Q.
rtl_sdr -f 1090000000 -s 2400000 -g 49.6 -n 288000000 kslc-1090-2m4-uc8.iq8
```

Notes: `-g 49.6` is the R820T max; if strong local aircraft overload the front end (lots of garbled frames), retry at `-g 40`. Add `-p <ppm>` if the dongle's crystal error is known. If an inline LNA needs bias-tee power, use an rtl-sdr-blog build with `-T`.

- [ ] **Step 2: Serve the file on the LAN** so the omnimodem host can pull it:

```sh
# In the directory holding the .iq8, on the Pi:
python3 -m http.server 8000
# Fetch from the dev host: curl -O http://<pi-ip>:8000/kslc-1090-2m4-uc8.iq8
```

- [ ] **Step 3: Get the reference decoder's number on the SAME file** (so both sides agree on methodology). With dump1090-fa or readsb:

```sh
readsb --ifile kslc-1090-2m4-uc8.iq8 --iformat uc8 --sample-rate 2400000 --quiet --stats
# or: dump1090 --ifile kslc-1090-2m4-uc8.iq8 --iformat UC8 --samplerate 2400000 --quiet --stats
# Record: total valid-CRC messages, and unique ICAO count.
```

- [ ] **Step 4: Commit a short clip for CI.** Cut the first ~300 ms (`head -c 1440000 kslc-1090-2m4-uc8.iq8 > adsb-clip.iq8`, 2.4 Msps × 0.3 s × 2 bytes = 1.44 MB) into `crates/dsp/tests/data/adsb-clip.iq8`. This is the regression fixture.

**Success:** we have (a) a full recording to iterate on, and (b) the reference frame/aircraft counts on it.

```rust
//! Offline ADS-B decode statistics for benchmarking against dump1090/readsb.

use super::message::ModeS;
use super::ppm::PpmDemodulator;
use std::collections::HashSet;

#[derive(Debug, Default, Clone)]
pub struct DecodeStats {
    pub frames: usize,       // valid-CRC frames
    pub unique_icao: usize,  // distinct ICAO addresses seen
    pub df17_18: usize,      // extended-squitter frames
}

/// Decode all frames in a magnitude slice at `samples_per_us` and summarize.
pub fn decode_stats(mag: &[f32], samples_per_us: usize) -> DecodeStats {
    let frames = PpmDemodulator::new(samples_per_us).scan(mag, true).0;
    let mut icao = HashSet::new();
    let mut df17_18 = 0;
    for f in &frames {
        let m = ModeS::new(&f.bytes);
        icao.insert(m.icao());
        if matches!(m.df(), 17 | 18) { df17_18 += 1; }
    }
    DecodeStats { frames: frames.len(), unique_icao: icao.len(), df17_18 }
}
```

- [ ] **Step 4: Run the test — expect PASS**

Run: `CARGO_INCREMENTAL=0 cargo test -p omnimodem-dsp --lib adsb::bench`
Expected: PASS.

use omnimodem_dsp::modes::adsb::bench::decode_stats;

fn main() {

}

## Task 2 (R1): Consume the reference 2.4 Msps rate

**Why:** readsb/dump1090-fa decode natively at 2.4 Msps; our demod assumes an integer 2 samples/µs. To compare fairly (and to be beatable), we must ingest 2.4 Msps. Start by resampling 2.4→2.0 with the existing `ComplexResampler`; native fractional-phase decode is folded into R3.

**Files:** Modify `crates/dsp/src/bin/adsb_bench.rs`, `crates/dsp/src/modes/adsb/bench.rs`.

- [ ] **Step 1: Test** — a 2.4 Msps magnitude stream carrying one known frame decodes after resampling to 2.0:

```rust
#[test]
fn decodes_after_resample_2m4_to_2m0() {
    use crate::frontend::resample::ComplexResampler;
    use crate::types::Cplx;
    use crate::modes::adsb::message::encode_identification;
    use crate::modes::adsb::ppm::PpmModulator;
    // Build a 2.0 Msps magnitude frame, upsample to 2.4 as complex (I=mag,Q=0),
    // then resample back to 2.0 and decode.
    let f = encode_identification(0x4CA123, "RYR1");
    let mag2m0 = PpmModulator::new(2).modulate_padded(&f, 8, 8);
    let up: Vec<Cplx> = ComplexResampler::new(2_000_000, 2_400_000, 8)
        .process(&mag2m0.iter().map(|&m| Cplx::new(m, 0.0)).collect::<Vec<_>>());
    let back: Vec<f32> = ComplexResampler::new(2_400_000, 2_000_000, 8)
        .process(&up).iter().map(|z| z.norm()).collect();
    assert!(super::decode_stats(&back, 2).frames >= 1);
}
```

- [ ] **Step 2: Run — expect FAIL** (helper not wired) then implement a `resample_to_2m0(iq: &[Cplx]) -> Vec<f32>` in `bench.rs` using `ComplexResampler::new(2_400_000, 2_000_000, 8)` then `.norm()`. Wire the bin to resample when invoked with `--in-rate 2400000`.

- [ ] **Step 3: Run — expect PASS.** Then run the bin on the real file at 2.4 Msps and record the number.

- [ ] **Step 4: Commit** `-m "adsb: ingest 2.4 Msps via ComplexResampler for the benchmark"`

**Success:** the harness produces a frame count on the real 2.4 Msps recording. **Record it in the design doc as the R0 baseline** next to the reference number. Everything after this is measured as a delta from R0.

## Task 3 (R2): Noise-floor-relative preamble detection

**Why:** the current `preamble_ok` requires *every* high slot > *every* low slot — great on the clean loopback, brittle off-air. dump1090 accepts a preamble when the four pulses stand above a **running noise floor** by a ratio and the quiet slots sit below it. `frontend::detector::EnvelopeDetector` already tracks an adaptive floor.

**Files:** Create `crates/dsp/src/modes/adsb/detect.rs`; modify `ppm.rs` to call it.

- [ ] **Step 1: Test** — a preamble buried in a raised noise floor (all slots + constant offset, pulses only modestly above) is still detected, whereas flat noise is not:

```rust
#[test]
fn detects_preamble_above_noise_floor() {
    // pulses at 3x floor, floor at 0.2; strict high>low would still pass here,
    // but add a single elevated quiet slot that must NOT veto detection.
    let det = PreambleDetector::new(2);
    let mut mag = vec![0.2f32; 64];
    for s in [0usize,2,7,9] { mag[s] = 0.6; }
    mag[4] = 0.35; // one noisy quiet slot below the pulses but above floor
    assert!(det.at(&mag, 0));
    assert!(!det.at(&vec![0.2f32; 64], 0)); // flat: no preamble
}
```

- [ ] **Step 2: Run — expect FAIL** (`PreambleDetector` undefined).

- [ ] **Step 3: Implement `detect.rs`** — a detector that computes the mean of the four pulse slots and the mean of the twelve quiet slots, plus a noise-floor estimate, and accepts when `pulse_mean >= floor * OPEN_RATIO && pulse_mean >= quiet_mean * MARGIN`. Keep `samples_per_us` handling identical to `ppm::slot_mag`. Constants (`OPEN_RATIO`, `MARGIN`) are tunables the benchmark will set.

- [ ] **Step 4: Run — expect PASS.** Add a `Demodulator`-level switch so `PpmDemodulator` uses `PreambleDetector` instead of `preamble_ok`; keep the old path behind a flag for A/B.

- [ ] **Step 5: Measure.** Run the harness on the real file with the old vs new detector; keep the new one only if frames increase without inflating false positives (spot-check that new ICAOs also appear in independently CRC-clean frames).

- [ ] **Step 6: Commit** `-m "adsb: noise-floor-relative preamble detection"` and record the delta in the design doc.

**Success:** measured frame count on the real recording rises vs R0; flat-noise test still yields zero.

## Task 4 (R3): Phase ensemble via the hydra

**Why:** at 2.4 Msps the bit centers fall on fractional sample phases; readsb tries multiple phase alignments. `ensemble::ParallelDemodulator` + `DedupWindow` run N demods over the same samples and union/dedup by `(content-hash, sample-offset)` — exactly this, and it is the graywolf-beats-Direwolf mechanism.

**Files:** Modify `ppm.rs` (parameterize the slot sampling by a sub-sample phase offset) and `adsb.rs` (build an ensemble of ADS-B slicers at phases `{-0.4,-0.2,0,+0.2,+0.4}` µs and thresholds).

- [ ] **Step 1: Test** — a frame whose bit centers are deliberately offset by a third of a slot (so the zero-phase slicer misses or errors) is recovered by the ensemble. Build the offending waveform by resampling a known frame with a fractional delay; assert single-phase `scan` yields 0 while the ensemble yields 1.

- [ ] **Step 2–4:** implement phase-shifted sampling in the slicer (add a `phase_offset: f32` to the demod that biases the slot center), construct the ensemble in `AdsbDemod`, dedup with `DedupWindow` (already generic over `Frame`). Run tests to green.

- [ ] **Step 5: Measure** on the real recording. Accept only if it beats R2. Log how many *extra* frames each phase contributes (drop phases that add nothing — no silent bloat).

- [ ] **Step 6: Commit** `-m "adsb: multi-phase slicer ensemble (hydra)"` + record delta.

**Success:** frame count ≥ readsb's on the recording (this is the phase most likely to reach parity), false positives still ~zero.

## Task 5 (R4): Soft-decision slicing (matched filter + decision-feedback AGC)

**Why:** hard first-half-vs-second-half comparison discards magnitude information. A matched filter for the 0.5 µs pulse (`frontend::fir`) plus `frontend::agc::DecisionFeedbackAgc` gives a soft per-bit metric that survives fades — graywolf's proven edge over Direwolf.

**Files:** Modify `ppm.rs` (matched-filter the magnitude before slicing; slice on the filtered soft metric), reuse `frontend::fir::design_lowpass`/FIR and `DecisionFeedbackAgc`.

- [ ] **Step 1: Test** — a fading frame (apply a slow amplitude ramp across the frame to the loopback waveform) that the hard slicer mis-slices is decoded correctly with the soft path. Assert bytes match the source frame.

- [ ] **Step 2–4:** implement, keep behind an A/B flag, tests green.

- [ ] **Step 5: Measure** on the real recording; keep only if it beats R3.

- [ ] **Step 6: Commit** `-m "adsb: matched-filter + DFB-AGC soft-decision slicing"` + delta.

**Success:** frame count ≥ R3 on the recording, especially on weak/distant aircraft.

## Task 6 (R5): CRC 1-bit repair with an ICAO roster

**Why:** dump1090/readsb recover frames with a single bit error using the CRC **syndrome** (each single-bit error maps to a fixed 24-bit syndrome). Uncontrolled, this manufactures false frames — so repairs and address-overlaid DFs (DF0/4/5/11/20/21, currently dropped) are accepted **only** when the resulting ICAO is on a recently-seen roster of confidently-decoded aircraft.

**Files:** Modify `crc.rs` (precompute a `syndrome -> bit position` table; `fix_single_bit(msg) -> Option<usize>`); create `roster.rs` (`Roster` = ICAO → last-seen sample offset, with an aging window); modify `adsb.rs` to consult them.

- [ ] **Step 1: Test (repair)** — flip one bit of the KLM1023 frame; `fix_single_bit` locates it and repairs to a zero checksum:

```rust
#[test]
fn repairs_single_bit_error() {
    // KLM1023 canonical frame (same hex() helper as the existing tests).
    let mut f = hex("8D4840D6202CC371C32CE0576098");
    f[5] ^= 0x08;
    assert_ne!(checksum(&f), 0);
    let pos = fix_single_bit(&mut f).unwrap();
    assert_eq!(checksum(&f), 0);
    assert_eq!(pos, 5 * 8 + 4);
}
```

- [ ] **Step 2: Test (roster gate)** — a repaired frame whose ICAO is NOT on the roster is rejected; the same frame after its ICAO is admitted (from a clean frame) is accepted. This is the false-positive guard.

- [ ] **Step 3–4:** implement the syndrome table (build once from the CRC generator: for each bit i, checksum of a message that is all-zero except bit i), the repair, the roster, and the acceptance policy in `AdsbDemod` (clean CRC → admit ICAO to roster; 1-bit-repaired or overlaid → accept only if ICAO on roster). Tests green.

- [ ] **Step 5: Measure** on the real recording. Report frames gained AND a false-positive estimate (repaired frames whose ICAO the reference never saw). Accept only if net-positive with FP under threshold.

- [ ] **Step 6: Commit** `-m "adsb: CRC single-bit repair gated by ICAO roster"` + delta.

**Success:** frame count rises again with false positives held near zero; DF11/DF4/5 now contribute where corroborated.

## Self-review notes

- Every phase R1–R5 is gated on *measured* improvement against Task 1's harder-than-loopback ruler; a phase that does not move the number on the real recording is reverted, not merged. That is the honesty guardrail — we do not claim "better" without the frame count to back it.
- Reused building blocks (verified present): `frontend::iq::u8_iq_to_cplx`, `frontend::resample::ComplexResampler`, `frontend::detector::EnvelopeDetector`, `frontend::agc::DecisionFeedbackAgc`, `frontend::fir`, `ensemble::{ParallelDemodulator, DedupWindow}`.
- Dependency order: Task 1 → 2 are prerequisites; 3/4/5 are independent experiments (any subset can land); 6 depends on 3 (roster wants the ensemble's clean frames first); 7 is last.
- Out of scope (separate plan): live daemon rtl_tcp wideband RX, `AircraftReport` proto, TUI flights table.
